# Canon and Side-Story Handling

Use this reference when the project is a prequel, sequel, side story, fanfiction, alternate route, or what-if line.

## First step

Define the relationship to parent canon:

- prequel
- sequel
- side story
- alternate route
- divergence / what-if
- inspired-by but not bound

## Required records

Document these in the project:

- parent work title
- canon scope actually considered authoritative
- divergence point
- spoiler boundary
- protected hooks that must remain unresolved
- protected world rules

## Audit priorities

Raise audit priority for:

- parent-canon contradiction
- future revelation leakage
- character foreknowledge
- premature payoff of parent hooks
- changes that accidentally invalidate the original timeline

## Good practice

Keep two categories separate:

### Parent-canon facts
Durable, protected facts from the original work.

### Local-story changes
What this derivative work adds, changes, or explores.

## Warning signs

- characters using knowledge not yet available in canon
- derivative plot solving the original story's mystery too early
- retcons hidden as "it was always like this"
- side characters becoming omniscient exposition channels
